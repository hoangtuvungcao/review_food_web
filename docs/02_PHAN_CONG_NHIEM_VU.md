# Phân công nhiệm vụ nhóm

## 1. Mô hình phân công

Nhóm gồm 03 sinh viên, mỗi thành viên đảm nhận một vai trò tương ứng với quy trình phát triển phần mềm tại doanh nghiệp.

## 2. Bảng phân công

| Thành viên | Vai trò | Nhiệm vụ chính | Sản phẩm phụ trách |
| --- | --- | --- | --- |
| Nguyễn Văn Trọng | Nhóm trưởng - Frontend Architect | Thiết kế cấu trúc HTML semantic, bố cục tổng thể, điều phối tiến độ, kiểm tra mã nguồn, tổng hợp báo cáo. | `index.html`, `about.html`, README, tài liệu báo cáo. |
| Lê Thuận | UI/UX Developer | Xây dựng giao diện, màu sắc, component, responsive PC/mobile, kiểm tra trải nghiệm người dùng. | `css/style.css`, giao diện danh sách, popup, form, footer. |
| Trần Văn Trường | JavaScript Engineer | Lập trình tương tác, xử lý dữ liệu, validation form, Leaflet map, localStorage, review và thêm địa điểm. | `js/main.js`, Leaflet marker, popup, form thêm quán và review. |

## 3. Mô tả chi tiết vai trò

### Nguyễn Văn Trọng - Nhóm trưởng / Frontend Architect

- Phân tích yêu cầu đề tài và xác định chức năng cần có.
- Xây dựng cấu trúc trang theo hướng semantic HTML.
- Thiết kế bố cục tổng thể bằng CSS Grid/Flexbox.
- Kiểm tra accessibility cơ bản: thẻ `label`, `alt`, nút đóng popup, form input.
- Quản lý GitHub repository và phê duyệt mã nguồn trước khi gộp.
- Tổng hợp báo cáo, slide và nội dung video demo.

### Lê Thuận - UI/UX Developer

- Thiết kế giao diện chính: header, sidebar danh sách, bản đồ, form thêm địa điểm, khung chi tiết.
- Tối ưu giao diện cho desktop và mobile bằng media queries.
- Thiết kế trạng thái hover, active, hidden, popup và drawer.
- Kiểm tra giao diện khi dữ liệu dài, ảnh upload và kích thước màn hình nhỏ.
- Đề xuất cải tiến trải nghiệm thao tác người dùng.

### Trần Văn Trường - JavaScript Engineer

- Tích hợp Leaflet map và marker địa điểm.
- Xử lý popup bản đồ, nút xem chi tiết và form đánh giá.
- Xử lý thêm địa điểm mới, bắt buộc upload ảnh/logo quán.
- Xử lý review có sao, bình luận và ảnh món ăn tùy chọn.
- Lưu và đọc dữ liệu từ `localStorage`.
- Kiểm tra lỗi nhập liệu và luồng tương tác chính.

## 4. Nguyên tắc phối hợp

- Mỗi thành viên làm việc trên phần được phân công.
- Thay đổi lớn cần báo nhóm trưởng trước khi gộp.
- Mã nguồn cần chạy được trên trình duyệt trước khi push.
- Cuối mỗi giai đoạn nhóm kiểm tra lại chức năng và giao diện.
