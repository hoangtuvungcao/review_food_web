# Kế hoạch và theo dõi tiến độ

## 1. Công cụ quản lý đề xuất

Nhóm có thể dùng một trong các công cụ sau:

- Jira: tạo task, phân công người thực hiện, theo dõi trạng thái.
- Trello: dùng bảng Kanban đơn giản.
- GitHub Projects: liên kết trực tiếp với repository.
- MS Project: lập timeline và phân bổ nguồn lực.

Trong phạm vi dự án này, nhóm mô phỏng bảng Kanban theo cấu trúc: `To Do`, `In Progress`, `Review`, `Done`.

## 3. Bảng Kanban mô phỏng

### To Do

- Đẩy repository lên GitHub.
- Deploy sản phẩm lên Netlify hoặc Vercel.
- Chuyển nội dung Markdown sang file `.docx`.
- Tạo file trình chiếu `.pptx`.
- Quay video demo 3-5 phút.
- Upload báo cáo lên Google Drive và nộp link LMS.

### In Progress

- Hoàn thiện tài liệu phân tích, phân công và đánh giá.

### Review

- Kiểm tra lại chính tả trong báo cáo.
- Kiểm tra đường dẫn deploy.
- Kiểm tra video demo có đủ chức năng chính.

### Done

- Xây dựng web tĩnh.
- Tích hợp Leaflet map.
- Tạo marker địa điểm.
- Tạo popup tóm tắt trên bản đồ.
- Tạo khung đánh giá chi tiết.
- Tạo chức năng thêm quán mới kèm ảnh/logo.
- Tạo chức năng review kèm ảnh món ăn tùy chọn.
- Tạo logo và favicon cho website.

## 4. Tiêu chí hoàn thành task

- Chức năng chạy được trên trình duyệt.
- Không có lỗi JavaScript cơ bản.
- Giao diện không vỡ trên desktop và mobile.
- Mã nguồn được lưu trong repository chung.
- Có tài liệu mô tả nhiệm vụ và kết quả.
