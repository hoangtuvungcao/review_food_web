# Phân tích yêu cầu dự án

## 1. Thông tin chung

- Tên đề tài: Cẩm nang ẩm thực đường phố.
- Chủ đề: Bản đồ vị trí các quán ăn ngon và hệ thống đánh giá từ người dùng.
- Nhóm: Nhóm 1.
- Thành viên:
  - Nguyễn Văn Trọng - Nhóm trưởng.
  - Lê Thuận.
  - Trần Văn Trường.

## 2. Yêu cầu bài tập

### Nhiệm vụ chung

Nhóm thực hiện phân tích và thiết kế hệ thống thông tin cho một website tĩnh mô phỏng hệ thống tra cứu, đánh giá địa điểm ẩm thực đường phố.

### Nhiệm vụ cá nhân

Mỗi thành viên được phân công vai trò cụ thể để mô phỏng quy trình làm việc thực tế tại doanh nghiệp. Nhóm cần có tài liệu theo dõi tiến độ bằng công cụ số như Jira, Trello, MS Project hoặc bảng Kanban mô phỏng.

### Quy trình nộp bài

- Mã nguồn đưa lên một GitHub repository chung.
- Sản phẩm được deploy trực tuyến bằng Netlify hoặc Vercel.
- Báo cáo gồm file `.docx`, file `.pptx` và video demo 3-5 phút.
- Báo cáo upload Google Drive, chia sẻ link và nộp link lên LMS/eLearning.

## 3. Đối chiếu với dự án hiện tại

| Nhóm yêu cầu | Trạng thái | Ghi chú |
| --- | --- | --- |
| Website tĩnh | Đã có | Dùng HTML, CSS, JavaScript, jQuery, Leaflet. |
| Bản đồ địa điểm | Đã có | Dùng Leaflet và OpenStreetMap. |
| Marker địa điểm | Đã có | Marker màu đỏ cho từng quán. |
| Danh sách quán | Đã có | Có tab gợi ý gần đây và top trending. |
| Tìm kiếm | Đã có | Tìm theo tên quán hoặc mô tả. |
| Popup trên bản đồ | Đã có | Có tên, logo, sao, mô tả rút gọn và nút xem chi tiết. |
| Xem chi tiết quán | Đã có | Có thông tin quán, mô tả và danh sách review. |
| Đánh giá quán | Đã có | Nhập tên, chọn sao, nhập bình luận. |
| Ảnh review | Đã có | Ảnh món ăn là tùy chọn. |
| Thêm quán mới | Đã có | Bắt buộc nhập tên, mô tả, ảnh/logo và chọn vị trí. |
| Lưu dữ liệu tạm | Đã có | Dùng `localStorage`. |
| Responsive PC/mobile | Đã có | CSS có media query cho màn hình nhỏ. |
| Phân công nhiệm vụ | Cần bổ sung tài liệu | Đã tạo file riêng trong thư mục `docs`. |
| Theo dõi tiến độ | Cần bổ sung tài liệu | Đã tạo file Kanban mô phỏng trong thư mục `docs`. |
| Đánh giá cuối dự án | Cần bổ sung tài liệu | Đã tạo file riêng trong thư mục `docs`. |
| Deploy Netlify/Vercel | Chưa thực hiện trong mã nguồn | Cần làm sau khi đẩy GitHub. |
| File `.docx`, `.pptx`, video demo | Chưa có | Các file `.md` hiện tại dùng làm nội dung gốc để chuyển sang báo cáo. |

## 4. Kết luận mức độ đáp ứng

Dự án hiện tại đã đáp ứng phần chính của sản phẩm web tĩnh và các chức năng mô phỏng hệ thống. Phần cần hoàn thiện tiếp theo là quy trình nộp bài: đẩy mã nguồn lên GitHub, deploy lên Netlify/Vercel, chuyển nội dung tài liệu Markdown sang `.docx`, tạo slide `.pptx` và quay video demo 3-5 phút.
